## Memory governance state

Two materially important AMAS states are preserved:

### State A
AMAS v1.2 + runtime-binding companion
- stronger external enforcement language
- single writable ledger
- runtime binding addendum
- preflight hard-fail
- repair / migration semantics

### State B
Rollback target to AMAS-1 v1.0
- no implicit memory load
- explicit rehydration required
- invalid cells blocked
- sealed memory only durable memory
- autoload / auto-assimilate behaviors disallowed

## Import posture

This mirror does not silently choose one state.
It preserves both and stays backward-compatible by keeping explicit-rehydration-safe semantics available as the common denominator.
