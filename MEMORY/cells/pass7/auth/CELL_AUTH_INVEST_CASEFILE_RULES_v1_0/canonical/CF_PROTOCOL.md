# CELL_AUTH_INVEST_CASEFILE_RULES_v1_0

## Purpose
Case file construction rules

## Canonical rules
- Case files should separate entities, events, claims, messages, and evidence.
- Source attribution must remain explicit.
- Unknowns and unresolved conflicts should remain visible rather than collapsed.
- Case structure supports later audit, retrieval, and public correction workflows.

## Notes
- Works with CCFLP and ICSA schema to produce repeatable investigative artifacts.
